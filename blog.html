Know The Differences

Clear your confusions between these keywords/methods

var, let, const : when to use what

The scope of var variable is functional scope. On the other hand, let and const variables have block scope. var variable can be updated and redeclared in the scope. On contrast, let variable can be updated but cannot be redeclared. And const variable cannot be updated or redeclared in the scope. var, let variable can be declared without initialization, but const variable cannot be declared without initialization.

Regular function vs Arrow function

Arrow function is a new feature introduced in ES6 that is a more compact syntax for writing function expressions. While both regular JavaScript functions and arrow functions work in a similar manner, there are certain differences between them. Regular function always starts with function keyword followed by optional function name, parameters in parentheses and syntex in curly braces. But arrow function use parentheses without function keyword. Rather it uses () => {} short syntax. Unlike regular functions, arrow functions do not have their own this. The value of this inside an arrow function remains the same throughout the lifecycle of the function and is always bound to the value of this in the closest non-arrow parent function. Arrow functions can never have duplicate named parameters, whether in strict or non-strict mode.

map(), forEach(), filter(), find() : Confuse what method to use with array!

All these methods are used to iterate over array and do some actions. However, they have different functionalities. For example, The forEach() method calls a function (a callback function) once for each array element. map() method does the same. But forEach() method does not return a new array, map() does. Also, map() method does not change the original array. The filter() method creates a new array with array elements that passes a test. Inside the callback function, there are some conditional syntax which is performed on each array element. The find() method returns the value of the first array element that passes a test function. Notice that filter() returns an array but find() returns a value.

What is "Template String"?

Template string use back-ticks (``) rather than the quotes ("") to define a string. It was first introduced in ES6. It has many advantages over using quotes. With template string, we can use both single and double quotes inside a string. It allows multiline strings. It provides an easy way to interpolate variables and expressions into strings. The method is called string interpolation. The syntax is - ${...}. In this way it allows us to use variables in strings. It also allows expressions in strings.
